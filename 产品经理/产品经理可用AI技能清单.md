# 产品经理可用 AI 技能 / 能力清单

面向日常工作的 AI 用法归纳，便于按需选用（含 Cursor Skill 思路与一般 AI 协作场景）。

![锁示意：保密与合规](./lock.png)

---

## 一、需求与文档类

| 场景 | 可用 AI 能力 | 简要说明 |
|------|----------------|----------|
| 需求澄清 | 追问式对话、假设列举 | 把模糊想法拆成用户故事、验收标准 |
| PRD / 需求说明 | 结构化写作、模板填充 | 按背景-目标-范围-流程-数据-风险输出初稿 |
| 用户故事 / 验收标准 | Given-When-Then、边界 case | 补全异常流、权限、空状态 |
| 竞品与调研摘要 | 长文提炼、对比表 | 多源信息合并成结论与差异点 |
|  release note / 更新说明 | 面向用户/对内两版语气 | 从 commit 或功能列表改写成可读文案 |

**可自建 Cursor Skill 方向：** 固定你们公司的 PRD 模板、术语表、评审检查清单，让 AI 按模板生成与自检。

| 场景 | 可用 AI 能力 | 简要说明 |
|------|----------------|----------|
| 数据口径说明 | 伪 SQL / 指标定义 | 与数据同学对齐前先写清「算什么、怎么算」 |

## 五、Cursor 中可落地的「Skill」类型（示例）

以下为 **Skill 文档可写成的主题**，便于团队统一提示词与产出格式：

1. **PRD-按我司模板生成** — 输入要点，输出带章节的标准 PRD。
2. **用户故事-INVEST 检查** — 对已有故事做完整性检查与改写建议。
3. **评审问题清单** — 输入功能描述，输出研发/设计/数据可能被问的问题。
4. **竞品对比表** — 输入要点或链接摘要，输出统一字段的对比表。
5. **上线检查清单** — 灰度、回滚、监控、公告、客服同步项。
6. **指标字典草稿** — 事件名、属性、计算方式、负责人占位。
7. **会议纪要 → 待办** — 固定格式：决议 / 待办 / 责任人 / 日期。

创建方式可参考 Cursor 官方 Skill 说明：在项目或用户目录添加 `SKILL.md`，写明何时调用、输入输出格式与约束。

## 六、使用注意（产品经理视角）

- **事实与数据**：市场数据、法规、竞品具体功能以核实为准，AI 适合起草与发散，不适合作为唯一信源。
- **保密**：客户名、未发布功能、内部数据勿粘贴到不可信环境；可用脱敏示例代替。
- **版权与合规**：对外文案、合同类内容需法务/合规复核。
